利用IPv6实现远程游戏联机

前言

最近想和在远方的朋友交流感情，于是选择了一起打游戏。除了一些自带服务器，可以方便联机的网游。我们也想玩一下我的世界、泰拉瑞亚等游戏。而这些游戏要实现联机就比网游困难多了。于是专门出本文来分享我的联机技术和经验。而其中最令我惊艳的还是IPv6技术，请听我细细讲来。

IPv6实现泰拉瑞亚（PC）联机

首先，你需要准备一台有充足流量的手机。然后对手机的流量进行设置：

  • 搜索APN（接入点名称）
  • 点击右上角三点，找到新建APN
  • 将APN协议和APN漫游协议设置为IPv6，其他参数仿照手机的默认APN修改，然后应用改APN
  • 打开移动热点让电脑连接热点

下面使用你的电脑进行操作：

  • 测试IPv6连接：测试链接 如果显示您的访问是IPv6即可进行下一步
  • 这一步开始会有操作风险，请谨慎操作！
  • 关闭电脑防火墙（设置IPv6防火墙有些困难，如果和朋友玩正常游戏的情况下，我们可以直接关闭防火墙）
  • 让你的朋友进行一遍上述操作
  • 打开命令提示行，输入ping 你的朋友的IPv6地址检查能否正常连接
  • 下载IPv6映射IPv4工具密码 hnpm
  • 打开工具进行主机映射，输入主机IPv6地址
  • 在另一人电脑上同样输入主机IPv6地址
  • 主机进行开服，客机使用127.0.0.1，26000端口加入游戏

如何在使用加速器的情况下与github建立ssh连接

前言

在使用hexo搭建博客的时候我遇到了这样一个问题:
$ ssh -T git@github.com
测试与GitHub的连接时，返回connection refused。此时我正在使用某加速器来加速访问GitHub。那么究竟为什么会导致失败呢？查阅发现加速器所经过的连接方式是不支持建立ssh连接的。因此我们需要进行一下改变。

对策

对策1

既然不能用那我就不用了呗。直接关闭加速器然后测试ssh。发现能够正常连接返回:
$ Hi USERNAME! You've successfully authenticated, but GitHub does not provide shell access.
但是话又说回来，本来使用加速器是因为访问比较慢，如果这样的话难不成我要一会开一会关加速器？

对策2

对于无法直接建立ssh连接的情况，我们的GitHub显然考虑到了这一点，因此有如下以HTTP模拟ssh的方法：
$ ssh -T -p 443 git@ssh.github.com
> Hi USERNAME! You've successfully authenticated, but GitHub does not provide shell access.
