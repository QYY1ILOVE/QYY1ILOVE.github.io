如何在使用加速器的情况下与github建立ssh连接

前言

在使用hexo搭建博客的时候我遇到了这样一个问题:
$ ssh -T git@github.com
测试与GitHub的连接时，返回connection refused。此时我正在使用某加速器来加速访问GitHub。那么究竟为什么会导致失败呢？查阅发现加速器所经过的连接方式是不支持建立ssh连接的。因此我们需要进行一下改变。

对策

对策1

既然不能用那我就不用了呗。直接关闭加速器然后测试ssh。发现能够正常连接返回:
$ Hi USERNAME! You've successfully authenticated, but GitHub does not provide shell access.
但是话又说回来，本来使用加速器是因为访问比较慢，如果这样的话难不成我要一会开一会关加速器？

对策2

对于无法直接建立ssh连接的情况，我们的GitHub显然考虑到了这一点，因此有如下以HTTP模拟ssh的方法：
$ ssh -T -p 443 git@ssh.github.com
> Hi USERNAME! You've successfully authenticated, but GitHub does not provide shell access.
